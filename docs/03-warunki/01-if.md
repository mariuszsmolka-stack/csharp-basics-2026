# Instrukcja warunkowa if

## Cel lekcji

Nauczysz się używać instrukcji `if`, aby wykonać fragment programu tylko wtedy, gdy warunek ma wartość `true`.

Po tej lekcji powinieneś umieć:

- wyjaśnić, po co stosuje się instrukcje warunkowe,
- zapisać prostą instrukcję `if`,
- rozpoznać warunek dający wynik `true` albo `false`,
- wskazać, które instrukcje należą do bloku `if`,
- zauważyć typowe błędy, na przykład użycie `=` zamiast `==`.

## 1. Krótkie wprowadzenie

Program bez instrukcji warunkowych wykonuje instrukcje po kolei, od góry do dołu.

Instrukcja `if` pozwala wykonać fragment kodu tylko w określonej sytuacji. Decyzja zależy od warunku.

Warunek to wyrażenie, które daje wynik:

- `true` - warunek jest spełniony,
- `false` - warunek nie jest spełniony.

Jeżeli warunek w `if` ma wartość `true`, blok kodu zostanie wykonany. Jeżeli warunek ma wartość `false`, blok zostanie pominięty.

## 2. Składnia if

Schemat instrukcji `if`:

```csharp
if (warunek)
{
    // instrukcje wykonywane, gdy warunek jest true
}
```

Najważniejsze elementy:

- warunek zapisujemy w nawiasach okrągłych `()`,
- instrukcje zależne od warunku zapisujemy w nawiasach klamrowych `{}`,
- jeśli warunek jest `false`, blok `if` zostaje pominięty.

## 3. Pierwszy przykład

```csharp
using System;

class Program
{
    static void Main()
    {
        int wiek = 18;

        if (wiek >= 18)
        {
            Console.WriteLine("Osoba jest pełnoletnia.");
        }
    }
}
```

Warunek `wiek >= 18` daje wynik `true`, ponieważ zmienna `wiek` ma wartość `18`.

Dlatego komunikat zostanie wypisany.

## 4. Przykład z warunkiem niespełnionym

```csharp
using System;

class Program
{
    static void Main()
    {
        int punkty = 30;

        if (punkty >= 50)
        {
            Console.WriteLine("Wynik jest wystarczający.");
        }

        Console.WriteLine("Koniec programu.");
    }
}
```

Warunek `punkty >= 50` daje wynik `false`, ponieważ `30` nie jest większe ani równe `50`.

Blok `if` zostanie pominięty.

Komunikat `"Koniec programu."` wypisze się zawsze, bo znajduje się poza blokiem `if`.

## 5. Warunki z operatorami

Operatory porównania były już omawiane w lekcji o operatorach. W instrukcji `if` używamy ich do budowania warunków.

Przykłady warunków:

```csharp
int x = 10;

x > 5
x < 5
x == 10
x != 10
x >= 10
x <= 7
x % 2 == 0
```

Warto pamiętać:

- `==` służy do porównania równości,
- `=` służy do przypisania wartości,
- `%` daje resztę z dzielenia,
- warunek `x % 2 == 0` sprawdza, czy liczba jest parzysta.

## 6. Zgadnij wynik warunku

Najpierw spójrz na przykłady z odpowiedziami.

```csharp
using System;

class Program
{
    static void Main()
    {
        int x = 10;

        Console.WriteLine(x > 5);       // true
        Console.WriteLine(x == 10);     // true
        Console.WriteLine(x != 10);     // false
        Console.WriteLine(x % 2 == 0);  // true
        Console.WriteLine(x < 0);       // false
    }
}
```

Teraz spróbuj samodzielnie przewidzieć wyniki.

```csharp
using System;

class Program
{
    static void Main()
    {
        int a = 7;
        int b = 12;

        Console.WriteLine(a > b);
        Console.WriteLine(a < b);
        Console.WriteLine(a + 5 == b);
        Console.WriteLine(b % 2 == 0);
        Console.WriteLine(a != 7);
    }
}
```

Odpowiedzi:

- `a > b` daje `false`,
- `a < b` daje `true`,
- `a + 5 == b` daje `true`,
- `b % 2 == 0` daje `true`,
- `a != 7` daje `false`.

## 7. Przykład z danymi z klawiatury - liczba dodatnia

```csharp
using System;

class Program
{
    static void Main()
    {
        Console.WriteLine("Podaj liczbę:");
        int liczba = int.Parse(Console.ReadLine());

        if (liczba > 0)
        {
            Console.WriteLine("Liczba jest dodatnia.");
        }
    }
}
```

Komunikat zostanie wypisany tylko wtedy, gdy użytkownik poda liczbę większą od `0`.

## 8. Przykład z parzystością

```csharp
using System;

class Program
{
    static void Main()
    {
        Console.WriteLine("Podaj liczbę:");
        int liczba = int.Parse(Console.ReadLine());

        if (liczba % 2 == 0)
        {
            Console.WriteLine("Liczba jest parzysta.");
        }
    }
}
```

Operator `%` oblicza resztę z dzielenia.

Liczba parzysta przy dzieleniu przez `2` daje resztę `0`.

Dlatego warunek `liczba % 2 == 0` sprawdza parzystość.

## 9. Przykład z progiem punktów

```csharp
using System;

class Program
{
    static void Main()
    {
        Console.WriteLine("Podaj liczbę punktów:");
        int punkty = int.Parse(Console.ReadLine());

        if (punkty >= 50)
        {
            Console.WriteLine("Próg punktowy został osiągnięty.");
        }
    }
}
```

Blok `if` wykona się tylko wtedy, gdy zmienna `punkty` ma wartość co najmniej `50`.

## 10. Przykład z temperaturą

```csharp
using System;

class Program
{
    static void Main()
    {
        Console.WriteLine("Podaj temperaturę:");
        double temperatura = double.Parse(Console.ReadLine());

        if (temperatura > 30)
        {
            Console.WriteLine("Uwaga: wysoka temperatura.");
        }
    }
}
```

Komunikat ostrzegawczy pojawi się tylko wtedy, gdy temperatura jest większa niż `30`.

## 11. Typowe błędy

### Błąd 1 - użycie = zamiast ==

Niepoprawnie:

```csharp
if (liczba = 10)
{
    Console.WriteLine("Liczba wynosi 10.");
}
```

Poprawnie:

```csharp
if (liczba == 10)
{
    Console.WriteLine("Liczba wynosi 10.");
}
```

`=` oznacza przypisanie wartości.

`==` oznacza porównanie równości.

### Błąd 2 - instrukcja poza blokiem if

```csharp
using System;

class Program
{
    static void Main()
    {
        int punkty = 30;

        if (punkty >= 50)
        {
            Console.WriteLine("Wynik jest wystarczający.");
        }

        Console.WriteLine("Ten komunikat wypisze się zawsze.");
    }
}
```

Tylko instrukcje w nawiasach klamrowych należą do bloku `if`.

Instrukcje poza blokiem wykonują się niezależnie od warunku.

### Błąd 3 - brak nawiasów klamrowych przy kilku instrukcjach

W C# czasem można spotkać `if` bez nawiasów klamrowych, gdy zależna jest tylko jedna instrukcja.

W tym kursie dla czytelności zawsze stosujemy nawiasy klamrowe przy `if`, nawet jeśli w bloku jest jedna instrukcja.

Czytelny zapis:

```csharp
if (punkty >= 50)
{
    Console.WriteLine("Wynik jest wystarczający.");
}
```

## 12. Zapowiedź kolejnej lekcji

W tej lekcji instrukcja `if` wykonuje kod tylko wtedy, gdy warunek jest spełniony.

W kolejnej lekcji pojawi się `if else`, czyli wybór jednej z dwóch możliwości.

## Zapamiętaj

- `if` sprawdza warunek.
- Warunek daje wynik `true` albo `false`.
- Blok `if` wykonuje się tylko dla `true`.
- Do porównania równości używamy `==`.
- Do przypisania używamy `=`.
- `%` pozwala sprawdzić resztę z dzielenia.
- Kod poza blokiem `if` wykonuje się niezależnie od warunku.
- W tym kursie zapisujemy nawiasy klamrowe przy każdym `if`.
